The Problem

Pre-Sales, Payment Reminder & Collection calling was outsourced to human call-centres — and it was breaking under its own volume:

1,000s of repetitive calls
Every campaign followed the same script across hundreds-to-thousands of borrowers, with near-zero variation between calls.
Most calls needed re-dials
Unpicked or inconclusive calls had to be re-dialled repeatedly until a clear borrower disposition was captured.
~8x cost via call-centres
NBFCs & SFBs paid human-caller salaries to run this volume — a prohibitively expensive, purely mechanical task.
My Role as AI-native IC PM
Script & Prompt Design
Designed call scripts and engineered system prompts for each of the 3 agent types — Pre-Sales, Payment Reminder, Collections — balancing personalization with reliability at scale.
Voice Model Selection
Evaluated and selected TTS models (Sarvam.ai, Smallest.ai) and Gemini 2.5 Flash for STT, based on voice-specific requirements distinct from text use-cases.
Re-Dial & Disposition Logic
Defined re-dial scheduling — up to 3 attempts at 2/4/8-hr gaps within TRAI-regulated windows — and the disposition rules that determine a "meaningful conclusion" per call.
Campaigns Module
Evolved the on-demand, 3-click scheduling feature into a campaigns module — bulk Excel upload and real-time CSV exports for 100s of borrowers per cycle.
The Solution — End-to-End Pipeline
Upload
Client uploads borrower list (Excel) for a PR / PC / PS campaign
Dial
Telephony (Exotel/Twilio) connects via Sarvam / Smallest TTS
Converse
In-house prompts drive the script live; Gemini 2.5 Flash for STT
Re-Dial
Up to 3x at 2/4/8 hrs within TRAI windows if inconclusive
Disposition
Promise-to-pay / partial / refusal → real-time CSV export
